[] 1.2 Настроить безопасный доступ к базе данных на хосте, включая firewall/SSH, резервное копирование и ограничения по источникам соединений, чтобы сайт уверенно работал на сервере.
[] 1.3 Определить и реализовать процесс деплоя веб-приложения на сервер с поддержкой HTTPS, доменного имени и автоматических обновлений чтобы пользователи могли создавать команды через браузер.
[] 2.1 Спроектировать схему БД для подписок (free и три тира), дополнив таблицы лимитами пользователей, статусом активации и привязкой к активирующим кодам.
[] 2.2 Разработать приватную консоль (не входящую в Docker-образ) только для меня, откуда можно выпускать криптографически защищённые коды активации и записывать их в БД с метаданными.
[] 2.3 Реализовать проверку кодов активации при регистрации/включении подписки, чтобы они были уникальны, не подделывались и активировали нужный уровень.
[] 2.4 Ввести логику проверки лимитов пользователей команды при каждом действии и отказываться от создания/добавления, если превышен соответствующий тир.
[] 2.5 Документировать, как лицензия и подписка проверяются и для публичного сайта, и для будущего локального Docker-режима, чтобы идея лицензирования неизменно сохранялась.
[] 3.1 Встроить обработку паролей при регистрации, включая сбор, хеширование (например bcrypt/argon2) и хранение только безопасных хешей в базе.
[] 3.2 Обновить потоки аутентификации, включая ограничение попыток входа, информативные ошибки, проверку хешей и обновление сессий при смене пароля.
[] 4.1 Переработать систему приоритетов задач: новый максимальный приоритет всегда равен текущему максимуму +1, и фронт/бэк синхронизировано проверяют это ограничение.
[] 4.2 Показывать чекбокс при удалении/завершении последней задачи уровня приоритета, дающий выбор удалить сам уровень или сохранить его для будущих задач.
[] 4.3 Обновить уведомления, чтобы баннеры вида «2 задачи требуют внимания» автоматически скрывались через таймаут и не перекрывали кнопки «календарь» и «выход».
[] 4.4 Расширить модель задач и формы, чтобы задачи создавались на временной диапазон (старт/конец) вместо одного дня, включая валидацию.
[] 4.5 Обеспечить корректное отображение диапазонных задач в календарях и уведомлениях, чтобы завершённые сегменты не вызывали старые оповещения.
[] 5.1 Подготовить Docker-образ с тем же приложением и лицензией, но способный запускаться полностью локально с одной командой, чтобы пользователи могли работать офлайн.
[] 5.2 Проверить, что лицензия/код активации работают и внутри Docker-версии, включая связку с приватной консолью и ограничение по числу пользователей команды.
